# exp004 - add hascabin feature

## 日付

2026-05-22

## コンペ名

titanic

## フェーズ

特徴量設計 / 実験記録

## 目的

真の baseline に `HasCabin` を追加したときの CV 差分を確認する。

## 仮説

`Cabin` 情報の有無は生存率と関係があり、`HasCabin` を足すと baseline より精度が上がる。

## 理由

EDA で `Cabin` 情報がある乗客とない乗客で生存率に大きな差が見えたため。  
細かい客室番号を使わなくても、`ある / ない` のフラグだけで有効な情報になる可能性がある。

## 変更内容

- `notebooks/exp004_HasCabin.ipynb` を作成した
- baseline の 7 列に `HasCabin` を追加した
- `Cabin.notna().astype(int)` で `HasCabin` を作った
- モデルや CV 設定は baseline から変えず、特徴量追加だけを比較した
- `submissions/exp004_hascabin_submission.csv` を生成して提出した

## 検証設計

- 分割方法: `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`
- 評価指標: Accuracy
- 比較条件: baseline と同じ `LogisticRegression`。違いは `HasCabin` の追加だけ

## 結果

- CV: mean `0.80247`, std `0.02448`
- CV fold scores: `0.80447`, `0.80899`, `0.76966`, `0.78652`, `0.84270`
- baseline CV mean: `0.79687`
- CV 差分: `+0.00560`
- LB: `0.76555`

## 気づき

- `HasCabin` 追加だけでも CV は baseline よりわずかに上がった
- ただし改善幅は `Title` よりかなり小さい
- Public LB は baseline と同じ `0.76555` で差が出なかった

## リスク

- `HasCabin` は `Pclass` や `Fare` と情報が重なっている可能性がある
- CV の改善幅が小さいため、過信しすぎない方がよい
- Public LB では差が出ていないため、採用判断は他の特徴との比較も踏まえる必要がある
