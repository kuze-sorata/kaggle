# exp007 - add sex x pclass interaction on top of title and familygroup

## 日付

2026-05-22

## コンペ名

titanic

## フェーズ

特徴量設計 / 実験記録

## 目的

現在のベスト構成 `Title + FamilyGroup` に `Sex x Pclass` を追加したときに、さらに改善するかを確認する。

## 仮説

`Sex` の効果は `Pclass` ごとに同じ強さではない。  
そのため、`SexPclass` を追加すると `Title + FamilyGroup` では取り切れない相互作用を表現でき、精度が上がるかもしれない。

## 理由

EDA でも `Sex` と `Pclass` はどちらも強い特徴だった。  
特に「女性は有利」「上位クラスは有利」という単独効果だけでなく、`female_1` と `female_3`、`male_1` と `male_3` のように組み合わせで差の大きさが変わる可能性があるため。

## 変更内容

- `notebooks/exp007_sex_pclass.ipynb` を作成した
- `Title` と `FamilyGroup` を含む現在のベスト構成に `SexPclass` を追加した
- `SexPclass = Sex + "_" + Pclass` のカテゴリ特徴を作った
- モデルや CV 設定は `exp006` から変えず、特徴量追加だけを比較した
- `submissions/exp007_sex_pclass_submission.csv` を生成した

## 検証設計

- 分割方法: `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`
- 評価指標: Accuracy
- 比較条件: `exp006` と同じ `LogisticRegression`。違いは `SexPclass` の追加だけ

## 結果

- CV: mean `0.83614`, std `0.00969`
- CV fold scores: `0.83799`, `0.82584`, `0.83146`, `0.83146`, `0.85393`
- `exp001` baseline CV mean: `0.79687`
- `exp003` Title CV mean: `0.82828`
- `exp006` Title + FamilyGroup CV mean: `0.82939`
- CV 差分 vs baseline: `+0.03927`
- CV 差分 vs `exp006`: `+0.00675`
- LB: `0.77511`

## 気づき

- `SexPclass` の追加で `exp006` より明確に改善した
- 特に `female_1`, `female_2`, `female_3` や `male_1`, `male_2`, `male_3` のように、性別と客室クラスの組み合わせごとに生存率差がかなり大きかった
- `Title` と `FamilyGroup` だけでは拾い切れない相互作用を補えている可能性が高い
- 今回の比較では、現時点の正式採用候補として最有力になった
- Public LB でも `0.77511` となり、`exp006` の `0.77272` から改善した

## リスク

- `Sex` と `Pclass` はどちらも強い特徴なので、交互作用追加が CV に対してだけ効いている可能性は残る
- Public LB ではまだ未確認のため、CV 改善がそのまま提出改善に繋がるとは限らない
- カテゴリを増やすため、別の特徴を足したときに冗長になる可能性がある
