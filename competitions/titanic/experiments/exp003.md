# exp003 - add title feature

## 日付

2026-05-22

## コンペ名

titanic

## フェーズ

特徴量設計 / 実験記録

## 目的

真の baseline に `Title` を追加したときの CV 差分を確認する。

## 仮説

`Name` から抽出した `Title` は、性別や年齢層の追加情報として効き、baseline より精度が上がる。

## 理由

EDA で `Mr`、`Mrs`、`Miss`、`Master` の生存率に差が見えたため。  
`Title` は `Sex` や `Age` に近い情報を持つが、それだけでは取り切れない違いも含んでいる可能性がある。

## 変更内容

- `notebooks/exp003_title.ipynb` を作成した
- baseline の 7 列に `Title` を追加した
- `Name` から敬称を抽出し、主要カテゴリに正規化した
- モデルや CV 設定は baseline から変えず、特徴量追加だけを比較した
- `models/exp003_title.joblib` を保存した
- `submissions/exp003_title.csv` を生成して提出した

## 検証設計

- 分割方法: `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`
- 評価指標: Accuracy
- 比較条件: baseline と同じ `LogisticRegression`。違いは `Title` の追加だけ

## 結果

- CV: mean `0.82828`, std `0.01308`
- CV fold scores: `0.82682`, `0.80899`, `0.84270`, `0.82022`, `0.84270`
- baseline CV mean: `0.79687`
- CV 差分: `+0.03141`
- LB: `0.76555`

## 気づき

- `Title` 追加だけで CV が明確に上がった
- 差分が 0.03 を超えており、偶然よりも有効な特徴量である可能性が高い
- `Name` の生文字列をそのまま使うのではなく、敬称だけ抜く方針は妥当そう
- Public LB は baseline と同じ `0.76555` で、CV 改善がそのまま LB 改善にはつながらなかった

## リスク

- `Title` は `Sex` や `Age` と情報が重なっており、過剰に複雑化すると過学習する可能性がある
- レア敬称のまとめ方によってスコアが変わる可能性がある
- Public LB では差が出ていないため、CV の改善を過信しすぎない
