# 実験ログテンプレート

このテンプレートは、各コンペの `experiments/` に共通で使うための基準です。

## `experiments/index.md`

```md
# 実験一覧

## exp001

- Date: 2026-05-22
- Phase: Baseline
- Change: first submit
- Result: CV TBD / LB TBD
- Status: WIP
```

## `experiments/expXXX.md`

```md
# expXXX - 実験タイトル

## 日付

YYYY-MM-DD

## コンペ名

<competition_name>

## フェーズ

問題理解 / ベースライン / CV / EDA / 特徴量設計 / モデル比較 / 提出 / 実験記録

## 目的

今回の実験で達成したいことを 1-2 文で書く。

## 仮説

何が効くと考えたかを書く。

## 理由

なぜその仮説を持ったかを書く。

## 変更内容

- 実装・設定・データ処理の変更点

## 検証設計

- 分割方法
- 評価指標
- 比較条件

## 結果

- CV:
- LB:

## 気づき

- 事実ベースの気づき

## リスク

- リーク、分布ずれ、評価の不安定さなど

## 次のアクション

- 次に試すこと
```

## 運用ルール

- `フェーズ` は標準フローのどこに属するかを明示する
- `experiments/index.md` は `## expXXX` ごとの短いブロックで書く
- 一覧では `Change` にその実験で何を変えたかを短く書く
- 一覧では `Result` に `CV` と `LB` をまとめて書く
- `変更内容` は事実だけを書く
- `気づき` と `次のアクション` を分ける
- 数値が未確定なら `TBD` と書く
